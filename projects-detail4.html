01. 왜 직접 만들었나

학과 재학 당시 '오픈소스 소프트웨어' 수업을 통해 Hugo와 AI로 포트폴리오를 만들었던 경험이 있습니다.
당시의 경험은 기술적 호기심을 자극했지만, 정해진 템플릿 안에서 움직이는 것은 마케터로서 저만의 색깔을 온전히 드러내기에 부족함이 느껴졌습니다.

그래서 이번에는 도구의 제약 없이 베이스부터 직접 구축하는 것을 택했습니다. Github Codespaces라는 개발 환경을 활용해 사이트의 기초를 다지고, AI를 단순한 생성 도구가 아닌 '기술적 조언자'로 리드하며 저만의 감각을 CSS로 구현해 냈습니다. 비전공자이지만 마케터에게 필요한 '기술적 이해도'와 '실행력'을 바탕으로, 나만의 브랜딩을 100% 투영할 수 있는 공간을 갖고 싶었습니다.

02. 무엇에 집중했나

Point 1

데이터와 기술을 이해하는 마케팅 감각

Tech-Friendly Branding

편리한 유료 툴 대신 Codespaces 환경에서 직접 코드를 핸들링하며, 브랜드 톤앤매너에 맞는 미니멀한 UI를 완성했습니다. AI와의 협업 과정에서 명확한 가이드라인을 제시하고 결과물을 조율하는 과정을 통해, 기술을 마케팅에 어떻게 녹여낼지 고민했습니다.

Deployment & Operation

Netlify를 활용한 실시간 배포 환경을 구축하여 마케팅 메시지가 즉각적으로 고객에게 전달되는 프로세스를 경험했습니다.

완전한 셀프 브랜딩

"이 사이트는 저와 함께 계속 성장할 공간입니다."

솔직히 고백하자면, 지금 보시는 이 포트폴리오는 제가 꿈꾸는 완벽한 모습의 절반도 채 되지 않습니다. 비전공자로서 직접 코드를 만지는 과정에서 밤을 새우기도 하고, 의도대로 구현되지 않아 수없이 고치기를 반복했습니다.

하지만 '부족함'은 곧 '성장할 여지'가 많다는 뜻이기도 합니다. Codespaces에서 코드를 한 줄씩 쌓아가는 이 집요함이 저에게는 가장 강력한 마케팅 무기라고 생각합니다. 앞으로 더 깊이 있는 데이터와 인터랙션을 추가하며, 이 아카이브를 완전히 저만의 색깔로 채워나갈 예정입니다.

지금의 미숙함이 훗날 어떤 확신으로 변할지, 그 과정을 끝까지 증명해내는 마케터가 되겠습니다.
이 기록은 멈추지 않고 계속 업데이트될 예정입니다.
